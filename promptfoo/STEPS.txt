Promptfoo in this repo — step by step
=====================================

1) Install Node.js (v18 or newer; v20+ recommended).

2) From the project root (same folder as package.json):
   npm install

3) Ensure `.env` exists with OPENAI_API_KEY (same as the Streamlit app).
   Copy from `.env.example` if needed.

4) Confirm Promptfoo is on your PATH (local node_modules):
   npm run verify:promptfoo

5) Run the eval suite (needs OPENAI_API_KEY in .env):
   npm run test:promptfoo
   (same as: npm run eval)

   This loads keys via Promptfoo’s --env-path .env and passes --no-write --no-cache
   so runs work reliably across machines (no dependency on the local Promptfoo DB).

Optional:
- npm run eval:record   — same tests but lets Promptfoo save results to its local DB (for `npm run eval:view`).
- npm run eval:view     — open the Promptfoo UI for recent runs (needs prior eval:record).

Upgrading Promptfoo:
- Newer promptfoo releases (0.121+) require Node ^20.20 or >=22.22. After upgrading Node:
  npm install promptfoo@latest --save-dev
  You can then drop --no-write if you want full DB features.

Files:
- promptfooconfig.yaml          — main config (prompt + provider + test file list)
- promptfoo/prompts/assistant.txt — prompt template with {{question}}
- promptfoo/tests/smoke.yaml    — example tests; add MMLU-style rows here later
